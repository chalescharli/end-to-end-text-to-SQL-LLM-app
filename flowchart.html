📋 Detailed Process Flow

1
User Input
User enters a natural language question in the Streamlit text input or selects from example questions.
2
Prompt + Question sent to Gemini
The app sends a predefined prompt (with table schema and examples) along with the user's question to Google Gemini model.
3
LLM generates SQL Query
Google Gemini analyzes the question and generates the corresponding SQL query based on the STUDENT table structure.
4
SQL Execution
The generated SQL query is executed against the SQLite database (student.db) using the red_sql_query function.
5
Results Displayed
Query results are displayed in a DataFrame format in Streamlit, with options to download as CSV. The query is also saved to history.

🛠️ Technology Stack Used

📱 Streamlit - UI Framework 🤖 Google Gemini - LLM 🗄️ SQLite - Database 🐍 Python - Language 📊 Pandas - Data Processing 🔑 python-dotenv - Environment Variables

🔗 What Gives What Output?

1
User Question (Text Input)
→ Gives: Natural language question
→ Goes to: Gemini API
2
Gemini API (LLM)
→ Takes: Prompt + User Question
→ Gives: SQL Query (string)
3
SQL Query
→ Takes: Generated SQL string
→ Goes to: SQLite Database
4
SQLite Database (student.db)
→ Takes: SQL Query
→ Gives: Query Results (list of tuples)
5
Streamlit App (Output)
→ Takes: Query Results
→ Gives: DataFrame Display + CSV Download + History
